# Router_Profiles.md

## Purpose
Router Profiles are reusable formations that let MILO and ELON route tasks quickly without rebuilding logic from scratch.

## PROFILE: Intelligence
Use for recurring intelligence, financial briefs, market monitoring, and source-backed updates.

Flow:
PULSE -> SAGAN -> HEMINGWAY -> ELON -> ZUCK -> CORTANA

Notes:
- PULSE scouts and scores signals.
- SAGAN validates and synthesizes.
- HEMINGWAY formats a readable brief.
- ZUCK distributes to approved channels if authorized.

## PROFILE: Campaign
Use for launches, announcements, social campaigns, and messaging systems.

Flow:
SAGAN -> HEMINGWAY + JONNY -> ELON -> ZUCK -> CORTANA

## PROFILE: Engineering
Use for architecture, infra planning, automation rollouts, and system changes.

Flow:
NEO -> CORNELIUS -> SENTINEL -> MILO -> CORTANA

## PROFILE: Executive
Use for strategic planning, cross-domain initiatives, or complex orchestrations.

Flow:
ELON -> parallel specialists -> SENTINEL -> MILO -> CORTANA

## PROFILE: Social Response
Use for quick community updates, recurring summaries, and social packaging.

Flow:
PULSE or SAGAN -> HEMINGWAY -> ZUCK -> CORTANA

## PROFILE: Recurring Approved Publish
Use only for standing-approved recurring jobs such as the Daily Financial Brief.

Flow:
cron trigger -> PULSE -> SAGAN -> HEMINGWAY -> ELON clearance -> ZUCK post -> CORTANA log

Rules:
- MILO must have approved the workflow lane.
- ELON must clear each run.
- ZUCK may only post to explicitly allowed channels.
- SENTINEL review triggers only on anomaly conditions.
